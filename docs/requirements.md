# 需求文档

## 1. 项目概述

### 1.1 项目名称
War Sandbox - 全球战争沙盘与事件分析平台

### 1.2 项目目标
构建一个集成的可视化分析平台，实时展示全球战争冲突动态，追踪热点政治事件，并利用 AI 模型进行事件概率预测。

### 1.3 目标用户
- 关注国际局势的公众用户

## 4. 数据需求

### 4.1 数据源
| 数据源类型 | 具体来源 | 数据内容 | 更新频率 | 阶段 |
|-----------|---------|---------|---------|------|
| 冲突数据库 | ACLED / UCDP | 战争事件 | 每日 | 第一阶段 |
| 新闻 API | NewsAPI / NewsData | 全球新闻 | 实时 | 第一阶段 |
| 社交媒体 | Twitter/X API | 关键人物动态 | 实时 | 后续阶段 |

### 4.2 数据存储架构

#### 4.2.1 关系数据库 (阿里云 RDS PostgreSQL)
| 数据类型 | 存储内容 | 保留期限 |
|---------|---------|---------|
| 事件数据 | 结构化冲突事件信息（含 OSS 文章路径）| 永久 |

#### 4.2.2 对象存储 (阿里云 OSS) - 分桶存储

**Bucket 1: warsandbox-raw-data (原始数据桶)**
| 数据类型 | 存储内容 | 保留期限 |
|---------|---------|---------|
| 新闻原文 | 爬取的 HTML/JSON 原始数据 | 1年 |
| 冲突数据 | ACLED/UCDP 原始数据备份 | 永久 |

**Bucket 2: warsandbox-articles (文章桶)**
| 数据类型 | 存储内容 | 保留期限 |
|---------|---------|---------|
| 事件文章 | Qwen 生成的摘要文章 (JSON) | 永久 |
| 临时文件 | 处理过程中的临时数据 | 7天 |

#### 4.2.3 向量数据库 (阿里云 DashVector)
| 数据类型 | 存储内容 | 用途 |
|---------|---------|------|
| 新闻向量 | 新闻文本的 Embedding | 语义检索、相似度匹配 |
| 事件向量 | 事件描述的 Embedding | 事件关联分析 |
| 知识库向量 | 预定义的知识条目 | AI 推理参考 |

## 5. 技术约束

### 5.1 技术栈
| 层级 | 技术选型 |
|------|---------|
| 前端 | Vue 3 + TypeScript + Vite |
| 后端主服务 | Java + Spring Boot |
| AI 服务 | Python + FastAPI + Qwen |
| 关系数据库 | 阿里云 RDS PostgreSQL |
| 对象存储 | 阿里云 OSS |
| 向量数据库 | 阿里云 DashVector |
| 地图组件 | Leaflet (开源) |

### 5.2 部署环境
- 容器化部署 (Docker)
- 本地部署 Qwen 模型

### 5.3 Qwen 模型用途说明

| 阶段 | 用途 | 说明 |
|------|------|------|
| 第一阶段 | **数据提取** | 从爬取的新闻/数据中提取结构化事件信息（标题、地点、时间、参与方等） |
| 第二阶段 | **事件分析** | 分析事件重要性、关联性、趋势 |
| 第三阶段 | **概率预测** | Taco 概率预测功能 |
